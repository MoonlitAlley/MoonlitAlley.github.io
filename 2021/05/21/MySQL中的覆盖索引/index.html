MySQL中的覆盖索引

Posted on 2021-05-21 In MySQL Valine:

昨天滴滴面试被问到这样一个问题：

假设存在一个person表， 表中有ID，name，sex三个字段， 然后name与sex是一个联合索引。
你能说明一下select name, sex from person where name = 'A';这个语句的查询过程中用到的索引么？

背景

当然，面试官不是直接就问我这个语句的，他的开始是想让我讲一下`覆盖索引`的定义及使用情况。我确实不了解这个`覆盖`的概念，就问了面试官“是非聚集索引查询过程中的回表操作么？”。
面试官没有直接否认，而是说：“没关系，概念这些不重要，咱们举一个具体的例子你来给我讲一下用到的索引吧”，于是有了上面这个问题。

知识背景

在分析这个问题之前，咱们需要先做一下基础知识准备。 可以通过以下几个问题逐渐过渡到面试官的问题上。

什么是回表查询

InnoDB的索引实现有两大类

  • 聚集索引（clustered index）
  • 普通索引（secondary index）
    有很多解释什么是聚集索引和非聚集索引的文章，这里只列一下两者的区别：
  1. 聚集索引的叶子结点存储行记录
    1. InnoDB必须有且只有一个聚集索引
    2. 如果表定义了PK，则PK就是聚集索引
    3. 如果表没有定义PK，则第一个not NULL unique列是聚集索引
    4. 否则InnoDB会创建一个隐藏的row-id作为聚集索引
  2. 普通索引的叶子结点存储主键值
